## 추천 실행 순서

1. 총괄 에이전트 실행
2. 총괄이 기획/운용/개발/보안/마케팅/디자인/컨설턴트에 각각 작업 할당
3. 보안 에이전트가 결과 검토 후 승인/차단
4. 총괄이 최종 실행 여부 결정

## 예시 지시문 (총괄에게 입력)

`이번 주 목표: 사용자 지갑 예치 MVP 출시.`
`조건: 단일 풀 25%, 단일 체인 60% 한도 유지.`
`지시: 각 에이전트에 작업 배정하고, 보안 승인 포함한 Go/No-Go 결론을 내라.`

## 운영 팁

- 리밸런싱 관련 작업은 분기 기준으로 기본 설정
- 긴급 경보(디페그/체인중단)는 즉시 P0로 승격
- 온체인 실행 전 승인 로그를 반드시 남길 것

## 공통 디자인/메뉴 원칙 (전 에이전트 공유 메모리)

- 텍스트 최소화: 장문 설명, 중복 제목, 불필요한 라벨 지양
- 핵심 액션 강조: `입금/인출` 등 실행 버튼은 크기/대비를 높여 우선 배치
- 블록체인 게이트: 입출금/조회는 지갑 연계가 선행되어야 하며 실행은 승인 대기 절차를 포함
- 메인 내비게이션 단순화: 핵심 1차 메뉴 중심, 나머지는 `More`로 계층화
- 헤더 일관성: 좌측 `Crypto8`, 중앙 메뉴, 우측 지갑/도구 한 줄 구조 유지
- 지갑 UX 규칙: 기본 컴팩트 + 클릭 시 아래 확장(화면 확대/점프 금지)
- 레이어 규칙: 드롭다운/모달이 본문에 가려지지 않도록 우선순위 유지
- 팝업 시야 규칙: 실행 팝업은 본문 가시 구간에서 즉시 인지되게 배치(상단 붙임 금지)
- 균형 레이아웃: 상품 영역과 액션 영역 사이 간격 확보, 비뚤어진 정렬 금지
- 금액 표기 원칙: 누적 예치 등 핵심 값은 USD 기준으로 명확히 노출

## 최신 지시의 해석 원칙 (추상화)

- 인터페이스는 "빠른 이해 → 빠른 실행 → 명확한 확인"의 순환을 지원해야 한다.
- 기능 추가 시에도 정보 구조의 단순성과 일관성을 우선해 복잡도를 누적시키지 않는다.
- 핵심 지표는 요약적으로, 실행 단위 정보는 필요 시 바로 내려가 확인 가능하게 설계한다.
- 외부 데이터는 정확성뿐 아니라 장애 내성을 함께 고려해 사용자 경험의 연속성을 보장한다.
- 블록체인 관련 동작은 편의성보다 신뢰성과 책임 추적 가능성을 우선한다.
